◎

Bounded Autonomy

Agents execute autonomously within guardrails. Humans control WHAT gets built and in what order. Quality gates halt execution on serious issues. A single .autopilot semaphore is the kill switch.

⬡

Domain Agnostic

Software development, data pipelines, asset generation, content campaigns. Battle-tested on 500+ user stories. The governance pattern works for anything large enough to need a backlog.

Execution Model

Three nested loops

Human intent flows inward through progressively tighter execution loops. Each loop has its own cadence, its own quality gate, its own halt condition.

Outer Loop

Human-Driven

Human controls the pipeline. Ideas enter through INBOX, get triaged into BACKLOG, refined through stages, and scheduled into TODO-Today.

INBOX.md → /triage → BACKLOG.md → Ideation → Refining → Ready → /workflow → TODO-Today.md
tasks queued

Inner Loop

Autopilot — 14 Steps

Autonomous execution. Reads next task, verifies DOR, implements with TDD, runs verification, code review, cleanup, commits, and loops.

Read task → Check semaphore → Verify DOR → TDD → Implement → Verify → Code review → Cleanup → Commit → DONE-Today → Loop
cleanup triggered

Cleanup Sub-Loop

Quality Gate

Automated quality enforcement. Runs greenlight, auto-fixes Low severity findings, halts on Medium+ for human review.

Run greenlight → Fix Low findings → Re-run greenlight → Medium+ found? → HALT → Human review
Pipeline

File-based pipeline, markdown all the way down

No database, no project management tool. Five markdown files govern the entire lifecycle. DOR gates entry, DOD gates exit. Everything is version-controlled.

Quality

Five-layer quality stack

No commit without greenlight. No bypass, no override, no exceptions. Quality is structural, not aspirational.

01
Definition of Ready (DOR)
Entry criteria. Every task must have user stories, acceptance criteria, and test files defined before implementation begins.
Entry
02
Greenlight Protocol
Automated validation suite. Runs all checks, type verification, pattern scanning, and integration tests in a single pass.
Automated
03
Cleanup Sub-Loop
Auto-fix Low severity findings. Halt on Medium+ for human review. Never downgrade, never bypass.
Enforcement
04
Definition of Done (DOD)
Exit criteria. App runs locally, existing functionality intact, tests pass, batch limits respected, atomic commit with audit trail.
Exit
05
KNOWN_PATTERNS.md
31 anti-patterns across 8 categories. Scanned on every commit. Re-introducing a known pattern is a quality regression, full stop.
Scan

Philosophy

Control WHAT and WHEN, not HOW

Shepherd separates the pipeline (deterministic) from execution (autonomous). Five layers, each with its own degree of freedom.

Pipeline Flow
Deterministic
Fixed order, no shortcuts. Graduation only through quality gates.
Inner Loop
Semi-deterministic
14 steps always the same, but agent decides how to execute each.
Routing
Algorithmic
Keyword scoring + capability matching. No LLM decides routing.
Execution
Autonomous
Agent chooses tools, files, approach. Skills provide guardrails only.
Kill Switch
Binary halt
.autopilot = pause → immediate stop. Medium+ findings → auto-pause.
LangGraph
More deterministic at the execution level — graph topology is fixed
CrewAI
More autonomous at the routing level — LLM delegates freely
Shepherd
Most rigid at the pipeline level, most free at the execution level
